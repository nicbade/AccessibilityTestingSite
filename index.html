Learn ARIA patterns with keyboard and screen readers

This site helps you practice WAI-ARIA design patterns such as accordions, tabs, menus, modals, and more—all optimized for keyboard-only navigation and screen reader users.

Patterns

Accordion Pattern

An accordion is a vertically stacked set of interactive headings that each contain a title and toggle associated content.

Accordion Accessibility Features

  • Tab / Shift+Tab: Move focus between accordion headers.
  • Enter or Space: Toggle the focused accordion panel.
  • Focus remains on the trigger element; screen readers announce state changes.
  • Hidden accordion content is also hidden from screen readers.

Breadcrumb Accessibility Features

  • <nav aria-label="Breadcrumb"> announces breadcrumb navigation.
  • Ordered list communicates hierarchical order.
  • The current page is unlinked, but can be marked with aria-current="page" when it is a link.
  • Focus indicators and hover states are visible for keyboard users.

Button Pattern

Button Accessibility Features

  • Use the native <button> element when possible (inherits full accessibility support).
  • Tab: Moves focus to the button.
  • Enter or Space: Activates the button.
  • Ensure visible focus indicators for keyboard users.

Carousel Pattern

Carousel Accessibility Features

  • Slides are structured as a semantic <ul> with each slide in a <li>.
  • Only the current slide is shown; inactive slides use the hidden attribute so they’re skipped by screen readers.
  • Navigation controls (Prev, Next, Pause) are real <button> elements with clear aria-label text.
  • Tab: Moves focus between controls.
  • Enter or Space: Activates the focused control button.

Modal Dialog Pattern

Simple Modal

This is a simple accessible modal dialog with some text and a close button.

Modal Accessibility Features

  • Tab and Shift+Tab: Cycle focus inside modal.
  • Esc: Closes modal.
  • role="dialog" and aria-modal="true" convey modal semantics.
  • Focus is returned to trigger after closing

Link Pattern

Clear, high-contrast links with default underlines and an external-link cue.

Internal link (inline): Go to Patterns.

External link (opens in a new tab): WAI-ARIA APG: Links (opens in a new tab) ↗

Visited link example (after you’ve clicked it): Example.com

Link Accessibility Features

  • Uses native <a> (Tab to focus; Enter to activate).
  • Underlines by default so color alone isn’t required (WCAG 1.4.1).
  • External links use target="_blank" with rel="noopener noreferrer" and an audible “opens in a new tab”.

Table Pattern

Semantic table with caption, headers, row headers, and optional sortable columns.

This table can be sorted by Name, Breed, or Age. Use Enter or Space on a column header to sort.

Adoptable Dogs
Chloe Golden Retriever 4 Available
Milo Border Collie 2 Fostered
Piper Labrador 7 Available
Zoe Beagle 5 Adopted

Accessibility Features

  • <caption> provides a clear title; column headers use scope="col", first cell in each row uses scope="row".
  • Sortable columns use real <button> elements; sort state is exposed on the <th> via aria-sort.
  • Visible focus rings come from your global :focus-visible styles; headers remain keyboard operable.
  • Overflow handling keeps the table usable on small screens without changing semantics.

How to test

  • Tab to a column header → focus ring is visible; press Enter/Space to sort.
  • Screen reader announces “ascending/descending” when sorting.
  • On small screens, horizontally scroll the table container; headers remain readable.
